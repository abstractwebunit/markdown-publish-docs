# За пару кликов

Самый быстрый путь от «папка с заметками» до «сайт в интернете». Ничего устанавливать не нужно.

## Вариант 1 — кнопка Netlify (самый простой)

[![Deploy to Netlify](https://www.netlify.com/img/deploy/button.svg)](https://app.netlify.com/start/deploy?repository=https://github.com/abstractwebunit/markdown-publish-template)

1. Жмёшь кнопку → входишь через GitHub (бесплатно).
2. Netlify сам создаёт копию шаблона в твоём GitHub и собирает сайт.
3. Через пару минут сайт живёт на `имя.netlify.app`.

Дальше просто кладёшь свои `.md`-файлы в папку `vault/` своего нового репозитория — сайт пересобирается сам при каждом изменении.

## Вариант 2 — кнопка Vercel

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/abstractwebunit/markdown-publish-template&project-name=my-notes&repository-name=my-notes)

То же самое, но на Vercel: кнопка → вход через GitHub → сайт на `имя.vercel.app`.

## Вариант 3 — шаблон + GitHub Pages (без сторонних сервисов)

Нужен только аккаунт GitHub. Подробные шаги: [[GitHub Pages]].

[Использовать шаблон →](https://github.com/abstractwebunit/markdown-publish-template/generate)

## Что внутри шаблона

Репозиторий [markdown-publish-template](https://github.com/abstractwebunit/markdown-publish-template) — это:

- `vault/` — твои заметки (несколько стартовых уже лежат);
- `markdown-publish.config.json` — настройки сайта ([[Конфигурация]]);
- готовые конфиги для GitHub Pages, Netlify и Vercel — потому кнопки и работают.

> [!note] Свои заметки уже в репозитории?
> Тогда шаблон не нужен — добавь workflow в свой репозиторий: [[GitHub Pages#Вручную в свой репозиторий]].
