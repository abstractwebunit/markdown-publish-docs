# 几次点击搞定

从“一个装笔记的文件夹”到“网站上线”的最快路径。什么都不用安装。

## 方式 1——Netlify 按钮(最简单)

[![Deploy to Netlify](https://www.netlify.com/img/deploy/button.svg)](https://app.netlify.com/start/deploy?repository=https://github.com/abstractwebunit/markdown-publish-template)

1. 点按钮 → 用 GitHub 登录(免费)。
2. Netlify 会自动在你的 GitHub 里创建模板副本并构建网站。
3. 几分钟后,网站就在 `站点名.netlify.app` 上线了。

之后只需把自己的 `.md` 文件放进新仓库的 `vault/` 文件夹——每次改动,网站都会自动重新构建。

## 方式 2——Vercel 按钮

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/abstractwebunit/markdown-publish-template&project-name=my-notes&repository-name=my-notes)

和上面一样,只不过在 Vercel:按钮 → 用 GitHub 登录 → 网站在 `站点名.vercel.app`。

## 方式 3——模板 + GitHub Pages(不依赖第三方服务)

只需要一个 GitHub 账号。详细步骤:[[GitHub Pages]]。

[使用模板 →](https://github.com/abstractwebunit/markdown-publish-template/generate)

## 模板里有什么

仓库 [markdown-publish-template](https://github.com/abstractwebunit/markdown-publish-template) 包含:

- `vault/`——你的笔记(已经放了几篇起步笔记);
- `markdown-publish.config.json`——网站设置([[配置]]);
- 为 GitHub Pages、Netlify 和 Vercel 准备好的配置——所以那些按钮才能直接工作。

> [!note] 笔记已经在自己的仓库里了?
> 那就不需要模板——把 workflow 加进自己的仓库即可:[[GitHub Pages#手动接入自己的仓库]]。
