# 如何更新网站

[[几次点击搞定|一键部署]]之后,你的笔记就放在**你自己的 GitHub 仓库**里(Netlify/Vercel 会在网站设置里显示仓库链接)。每次 commit,网站都会**自动重新构建**——想怎么更新都行:

## 方式 1——直接在浏览器里(什么都不用装)

1. 打开你的仓库 → `vault/` 文件夹。
2. 点开笔记 → 铅笔图标 **Edit** → 修改 → **Commit changes**。
3. 新建笔记:**Add file → Create new file** → 文件名形如 `我的笔记.md`。

大约 1–2 分钟后,改动就会出现在网站上(如果没看到——强制刷新,`Ctrl+Shift+R`)。

## 方式 2——通过 Obsidian

```bash
git clone https://github.com/你的用户名/你的仓库名
```

在 Obsidian 里:**Open folder as vault** → 选择克隆目录里的 `vault/` 文件夹。照常写笔记,然后 commit 和 push:

```bash
git add -A && git commit -m "notes" && git push
```

> [!tip] 自动同步
> 插件 **obsidian-git** 可以按计划自动 commit 和 push——这样“刚写完”网站就更新了。

## 方式 3——浏览器里的 VS Code

在 GitHub 上打开仓库,按 `.`(句点)键——会打开 github.dev 网页编辑器:一次改很多文件很方便。

## 给网站和仓库改名

Netlify 创建的仓库会带一个随机后缀(形如 `markdown-publish-template-61e7d`)——这是正常的,而且可以改:

- **页头里的网站名称:**`markdown-publish.config.json` 里的 `siteName` 键([[配置]])。
- **网站地址:**Netlify → Site configuration → **Change site name**(会变成 `你的名字.netlify.app`)。Vercel 是 Settings → Domains。
- **仓库名:**GitHub → Settings → **Rename**。Netlify/Vercel 不会坏——GitHub 会自动重定向旧名字。

## 重新构建要等多久

第一次构建最慢(托管平台要下载构建器)。之后有缓存会更快,通常 1–2 分钟。状态可以在 Deploys 标签页(Netlify)/ Deployments(Vercel/GitHub)里看到。
