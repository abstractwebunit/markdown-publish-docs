# Bienvenido

**markdown-publish** convierte una carpeta de notas (un vault de Obsidian o simplemente Markdown) en un sitio estático rápido — uno como este. Este sitio está construido con él a partir de archivos `.md` normales.

> Abre [[En un par de clics]] — ahí hay tres botones y el resto va solo.

## Qué sabe hacer

- **[[Búsqueda]]** — instantánea, funciona directamente en el navegador (pulsa `Ctrl+K`).
- **[[Grafo de enlaces]]** — un mapa interactivo de las notas, como en Obsidian (enlace «Graph view» a la izquierda).
- **[[Tableros canvas]]** — los archivos `.canvas` se renderizan como tableros interactivos: aquí tienes un [[Mapa de guías.canvas|ejemplo en vivo]].
- **[[Agentes de IA (WebMCP)|Agentes de IA]]** — el sitio entrega las notas a los agentes de IA mediante WebMCP.
- `[[wikilinks]]`, enlaces markdown normales, embeds, callouts, etiquetas, notas al pie, resaltado de código.
- Tema oscuro/claro, versión móvil, vista previa de la nota al pasar el cursor por un enlace.
- SEO de serie: sitemap, robots.txt, llms.txt, tarjetas OG, JSON-LD.
- Caracteres no ASCII en las URL de las páginas (cirílico, acentos) — como en este sitio.

## Cómo publicar tus notas

| Camino | Dificultad | Guía |
|---|---|---|
| Botón de Netlify / Vercel | 2 clics | [[En un par de clics]] |
| Tu vault existente → GitHub | 10 minutos, sin terminal | [[Tu vault existente en GitHub]] |
| Plantilla + GitHub Pages | 3–4 clics | [[GitHub Pages]] |
| Tu propio repositorio + workflow | para quien conoce git | [[GitHub Pages]] |
| Cloudflare Pages | un par de minutos | [[Cloudflare Pages]] |
| En local con la CLI | terminal | [[CLI]] |

¿Ya publicaste? El siguiente paso es [[Cómo actualizar el sitio]]. Todos los ajustes (nombre, idioma, qué notas publicar) están en [[Configuración]].
