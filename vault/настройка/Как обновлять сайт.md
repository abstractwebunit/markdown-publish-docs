# Как обновлять сайт

После [[За пару кликов|деплоя кнопкой]] твои заметки живут в **твоём GitHub-репозитории** (Netlify/Vercel показывают ссылку на него в настройках сайта). Сайт пересобирается **сам при каждом коммите** — обновлять можно как удобно:

## Способ 1 — прямо в браузере (без установки чего-либо)

1. Открой свой репозиторий → папка `vault/`.
2. Кликни заметку → карандаш **Edit** → правь → **Commit changes**.
3. Новая заметка: **Add file → Create new file** → имя вида `Моя заметка.md`.

Через ~1–2 минуты изменения на сайте (если не видно — жёсткая перезагрузка, `Ctrl+Shift+R`).

## Способ 2 — через Obsidian

```bash
git clone https://github.com/ТВОЙ-НИК/ТВОЙ-РЕПОЗИТОРИЙ
```

В Obsidian: **Open folder as vault** → выбери папку `vault/` внутри клона. Пишешь как обычно, потом коммит и push:

```bash
git add -A && git commit -m "notes" && git push
```

> [!tip] Автосинхронизация
> Плагин **obsidian-git** умеет коммитить и пушить сам по расписанию — тогда сайт обновляется «как только написал».

## Способ 3 — VS Code в браузере

Открой репозиторий на GitHub и нажми клавишу `.` (точка) — откроется веб-редактор github.dev: удобно править сразу много файлов.

## Переименовать сайт и репозиторий

Netlify создаёт репозиторий со случайным суффиксом (вида `markdown-publish-template-61e7d`) — это нормально, и это лечится:

- **Имя сайта в шапке:** ключ `siteName` в `markdown-publish.config.json` ([[Конфигурация]]).
- **Адрес сайта:** Netlify → Site configuration → **Change site name** (будет `твоё-имя.netlify.app`). У Vercel — Settings → Domains.
- **Имя репозитория:** GitHub → Settings → **Rename**. Netlify/Vercel не сломаются — GitHub перенаправляет старое имя автоматически.

## Сколько ждать пересборку

Первая сборка самая долгая (хостинг скачивает сборщик). Дальше — быстрее за счёт кэша, обычно 1–2 минуты. Статус виден во вкладке Deploys (Netlify) / Deployments (Vercel/GitHub).
